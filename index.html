Features

Overview of Projects and Releases

Create or get invited into an organization

In MLCenter we have the concept of an organization, which can be created by any user.

Organization Members

The owner of an organization can invite other users and assign them one of the following roles: member, admin or owner.

Projects

In MLCenter the organization owns the projects and only the owner can delete them. However a member of an organization can create as many projects as they want.

Lifecycles

The other component owned by an organization are the lifecycles, which are later on assigned to the Model Registry within a project.

Lifecycles Stages

The owner of the organization can define the stages within a lifecycle and assing access roles(same roles as above) for each stage to restrict which user can deploy models between stages.

Experiments

A member of an organization can create as many experiments as they through the MLCenter python package and keep track of the model hyperparameters, metrics and artifacts.

Experiment Vizualizer

The user can visualize the experiment hyperparameters, metrics and artifacts right inside the platform, and also download the artifacts.

Model Registry / Deployments

Few steps back we were talking about lifecycles stages, following on that a user can deploy a model between stages based on their role.

Open API

The python package is just a client using the Back-End API of the MLCenter platform, but you can also use the API directly with your credentials.
